# 금요일 30분 주간 회고 루틴 — 번아웃 없이 3년째 유지한 생산성 시스템

**📅 작성일**: 2026년 4월 7일

> [!NOTE]
> :bulb: 회고는 중요하다는 건 누구나 안다. 문제는 "매일 일기"나 "월간 포트폴리오 정리"처럼 부담이 큰 루틴은 3주 만에 무너진다는 점이다. 이 글은 지난 3년간 꾸준히 유지해 온 **주 1회 30분, 세 가지 질문**으로 구성된 미니멀 회고 시스템을 공개한다. 과도한 구조나 도구 없이도 장기 성과가 자동으로 누적되는 저비용 고효율 루틴이다.

## 왜 "주간"이 균형점인가

회고 주기별 특성:

| 주기 | 장점 | 단점 |
|------|------|------|
| 일간 | 기억 선명 | 피로도 높음, 2주 내 포기 |
| **주간** | **균형** | **없음 (실천 시)** |
| 월간 | 부담 낮음 | 세부 기억 휘발, 반복 학습 약함 |
| 분기 | 전략적 뷰 | 전술 조정 늦음 |

주간 회고는 **기억이 생생한 동시에 반복 부담이 적은** 유일한 구간이다. 연간 52회 루프로 복리 효과도 크다.

## 세 가지 질문 — 이것만 있으면 된다

### 질문 1: 이번 주 한 일은?

**성취 기록**. 주관적 감상이 아니라 사실 나열.

```
- 결제 시스템 리팩토링 PR 3건 머지
- 팀 위키에 API 가이드 추가
- 신입 온보딩 문서 업데이트
- Rust 책 1-3장 완독
```

> [!TIP]
> "뭘 했는지 기억이 안 나"는 회고 실패의 가장 흔한 원인이다. 캘린더·Jira·Git 커밋 로그를 30초만 훑으면 사실 기록은 금방 끝난다.

### 질문 2: 이번 주 배운 것은?

**학습 정착**. 단순 정보가 아니라 **다음 주에도 써먹을 통찰** 수준으로 기록.

```
- EXPLAIN ANALYZE의 Rows Removed가 1천만 넘으면
  거의 확실히 인덱스 문제 (실측 확인)
- PR 리뷰는 오전 10시 이전에 보내야 당일 응답률 90%
- 퇴근 전 30분 산책이 수면 질에 가장 큰 영향
```

### 질문 3: 다음 주 3가지는?

**방향 설정**. 우선순위가 뚜렷해야 다음 주에 길을 잃지 않는다.

```
1. 결제 리팩토링 마무리 (비즈니스 영향 최대)
2. Rust 4장 완독 (학습 관성 유지)
3. 팀 리드와 1:1 (분기 목표 정렬)
```

네 개 이상 쓰지 않는다. **세 개를 넘는 순간 전부 흐려진다.**

## 루틴 운영 실전 팁

### 팁 1: 고정 시간 슬롯

> [!IMPORTANT]
> **"시간 나면 하겠다"는 루틴은 반드시 실패한다.** 금요일 오후 5시처럼 명확한 슬롯을 캘린더에 반복 일정으로 박아 두자. 회의 요청이 와도 "회고 루틴입니다"라고 거절할 수 있을 만큼 고정되어야 한다.

### 팁 2: 같은 템플릿 재사용

Notion/Obsidian 템플릿으로 매주 복제:

```
# Week N — YYYY-MM-DD
## 이번 주 한 일
-
## 이번 주 배운 것
-
## 다음 주 3가지
1.
2.
3.
```

양식이 같아야 주간 비교가 가능하다. 시각적 피드백이 지속 동기의 핵심.

### 팁 3: 30분 타이머

30분을 넘기는 순간 다음 주에 빼먹고 싶어진다. 타이머를 걸고 **끝났으면 그냥 저장하고 퇴근하라.** 완벽주의는 지속성의 적.

## 6개월 후 변화

3년 전 시작했을 때와 지금의 차이:

| 항목 | 시작 전 | 3년 후 |
|------|---------|--------|
| "올해 뭐 했지?" 질문 | 즉답 불가 | 회고 검색 30초 |
| 이직 포트폴리오 준비 | 2~3주 소요 | 1일 내 정리 |
| 분기 목표 리뷰 | 주관적 체감 | 데이터 기반 |
| 동일 실수 반복 | 자주 | 거의 없음 (메모로 방지) |

## 흔한 실패 원인

| 원인 | 해결 |
|------|------|
| 템플릿이 너무 상세 | 세 질문으로 압축 |
| 완벽한 문장을 쓰려 함 | 불릿 + 키워드로 충분 |
| 공유 부담 (팀 공개 등) | 개인용으로 시작 |
| 놓친 주가 있을 때 죄책감 | 스킵하고 다음 주 재개 |

> [!WARNING]
> 한두 번 빼먹었다고 **포기하지 마라.** 3년간 유지한 나도 연 5~6주는 건너뛴다. 완벽한 연속보다 "80% 지속률"이 훨씬 건강하다. 누락 주는 공란으로 두고 다음 주에 새로 시작.

## 선택지: 팀 단위 확장

개인 루틴이 정착됐다면 팀에 도입을 고려할 만하다. 단, 주의사항:

- 자발적 참여여야 함 (강제 시 형식화)
- 비공개 섹션 허용
- 리더가 먼저 본인 회고 공유

## 체크리스트

- [ ] 고정 시간 슬롯을 캘린더에 등록했는가
- [ ] 세 질문만 있는 템플릿을 준비했는가
- [ ] 30분 타이머를 쓰는가
- [ ] 완벽주의를 내려놓고 불릿으로 쓰는가
- [ ] 누락 주에 대한 죄책감 대신 재개 규칙을 정했는가

> [!CAUTION]
> 회고 루틴을 **성과 측정 도구로만** 사용하면 스스로를 채찍질하는 방향으로 변질된다. "한 일"만 늘리려다 "배운 것"을 소홀히 하는 순간 시스템이 망가진다. 세 질문의 균형을 지키자.

## 정리

생산성 시스템을 설계할 때 흔히 빠지는 함정은 **정교하게 만들수록 좋을 것**이라는 착각이다. 3년간 검증한 결과, 회고의 본질은 도구나 구조가 아니라 **반복**이다. 세 질문·30분·금요일 — 이 세 가지 제약을 지키는 것만으로 충분하다. 오늘 퇴근 전 캘린더에 반복 일정 하나만 등록하자. 6개월 뒤의 당신이 고마워할 것이다.
