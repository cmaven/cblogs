# 개발자를 위한 노트 시스템 비교

## 왜 노트 시스템이 중요한가

개발자는 매일 새로운 기술, 디버깅 과정, 아키텍처 결정을 접합니다. 체계적인 노트 시스템이 없으면 같은 문제를 반복해서 해결하게 됩니다.

## 주요 노트 앱 비교

### Obsidian

- **저장**: 로컬 마크다운 파일
- **강점**: 양방향 링크, 그래프 뷰, 플러그인 생태계
- **코드 지원**: 마크다운 코드 블록, Mermaid 다이어그램

```markdown
# 오늘 배운 것
- [[TypeScript]] 제네릭에서 extends 키워드 활용
- [[Docker]] 멀티스테이지 빌드 최적화
```

### Notion

- **저장**: 클라우드
- **강점**: 데이터베이스, 팀 협업, 풍부한 블록 타입
- **약점**: 오프라인 지원 부족, 마크다운 내보내기 불완전

### Logseq

- **저장**: 로컬 마크다운/Org-mode
- **강점**: 아웃라이너 기반, 일일 저널, 양방향 링크
- **약점**: 긴 문서 작성에 불편

## 추천 워크플로우

| 용도 | 추천 도구 |
|------|----------|
| 개인 지식 관리 | Obsidian |
| 팀 문서화 | Notion |
| 일일 로그 | Logseq |
| 빠른 메모 | Apple Notes / Google Keep |

## 나의 선택: Obsidian

1. 로컬 파일이라 Git으로 백업 가능
2. 플러그인으로 무한 확장
3. 마크다운이라 이식성 높음
4. 빠른 검색 속도

어떤 도구를 선택하든 **꾸준히 기록하는 습관**이 가장 중요합니다.
